Privacy Policy

Last updated: January 2026

TimeDraft is a Chrome extension that helps users generate and insert calendar availability into Gmail drafts. This privacy policy explains how TimeDraft accesses, uses, stores, and protects Google user data.

Data Accessed

TimeDraft accesses the following Google user data:

  • Google Calendar data (read-only), including event start and end times, to determine when a user is available.
  • Basic account information provided by Google during OAuth authentication (such as an email address or profile identifier).

How Data Is Used

Google user data is used solely to provide the core functionality of TimeDraft. Specifically:

  • Calendar data is processed to calculate available time windows.
  • Generated availability is displayed to the user for insertion into Gmail drafts.

TimeDraft does not use Google user data for advertising, analytics, marketing, or training machine learning models.

Data Sharing

TimeDraft does not sell, rent, or share Google user data with third parties.

Data is only transmitted between the extension and Google APIs as required to retrieve calendar availability.

Data Storage & Security

TimeDraft does not store Google Calendar event data on external servers.

  • OAuth access tokens are stored locally in the user’s browser using Chrome extension storage.
  • All communication with Google APIs occurs over secure HTTPS connections.

Data Retention

TimeDraft retains Google user data only for as long as necessary to provide its functionality.

  • Calendar data is processed in real time and is not persistently saved.
  • OAuth tokens remain stored locally until the user disconnects the extension or removes it.

Data Deletion

Users can delete all data associated with TimeDraft at any time by:

  • Removing the TimeDraft Chrome extension from their browser
  • Revoking TimeDraft’s access in their Google Account under Security → Third-party access
